# 납품 유효기간 관리 앱

제품을 큰 대표제품명으로 묶고, 세부 모델/규격/옵션 차이는 `세부모델/규격`, `LOT번호`, `기타작성칸`으로 관리하는 웹 기반 초안입니다.
데이터 저장소는 하나로 두되, 화면에서는 `소모품`과 `장비`를 별도 사이트처럼 전환해서 볼 수 있게 구성했습니다.

## 사용 방식

1. `index.html`을 브라우저에서 엽니다.
2. 사이트가 열리면 온라인 저장소의 데이터를 자동으로 불러옵니다.
3. 좌측에서 `소모품` 또는 `장비` 모드를 선택합니다.
4. 조회, 분석, 보고서 확인은 누구나 할 수 있습니다.
5. 데이터 입력/업로드/온라인 저장은 관리자 비밀번호 확인 후 진행합니다.
6. `단건 입력`에서 신규 납품, 교환, 회수, 샘플/데모 이력을 추가합니다.
   - 입력 데이터에는 `분류`가 함께 저장됩니다.
   - 대표제품명은 선택한 분류에 맞는 항목만 먼저 표시됩니다.
   - `연결그룹`은 선택 입력입니다. 장비와 소모품을 같은 사용군으로 묶어두고 싶을 때 적습니다.
7. `대량 입력`에서 양식을 다운로드한 뒤 Excel에서 작성하고, CSV/XLSX 파일을 다시 올립니다.
   - 파일 안에서 소모품/장비 데이터를 섞어 올릴 수 있습니다.
   - `분류` 컬럼에 `소모품` 또는 `장비`를 넣으면 됩니다.
   - 파일 업로드, 미리보기 적용, 온라인 저장은 비밀번호 `5802` 입력 후 가능합니다.
   - `현재 데이터` 버튼으로 지금까지 입력된 전체 데이터를 엑셀 파일로 내려받을 수 있습니다.
8. `분석하기`에서 아래 3가지 보고서를 확인합니다.
   - 제품별 단가 확인
   - 병원 납품 시기 예측
   - 마지막 납품 시점 + 유효기간 + 단가 확인
9. 분석 화면 하단에서 연결 이력을 확인합니다.
   - 소모품 모드에서는 같은 병원에 납품된 장비 이력이 보입니다.
   - 장비 모드에서는 같은 병원에 납품된 소모품 이력이 보입니다.
10. `1장 보고서 인쇄`를 누르면 현재 선택한 분류 기준으로 A4 한 장짜리 보고서가 출력됩니다.

## 만료/교환 우선 확인 기준

현황 화면의 `만료/교환 우선 확인`은 모든 만료 데이터를 보여주는 목록이 아니라, 선납 정산 누락 위험이 있는 건만 우선 표시합니다.

- 납품일이 오늘 기준 최근 1년 이내인 건
- 유효기간이 90일 이내이거나 이미 만료된 건
- 회계일자가 비어 있는 미정산 건
- 간납/대학병원 관리대상 키워드에 해당하는 병원 또는 간납처 건
- 수량이 양수이고 `신규납품`인 건

회계일자가 있는 건은 이미 사용 및 정산이 진행된 건으로 보고 우선 확인 목록에서 제외합니다.
최근 납품 목록도 오늘 기준 최근 1년 이내 데이터만 표시합니다.

## 데이터 수정/삭제

현황 화면의 `데이터 수정/삭제`에서 병원명, 제품명, LOT, 메모로 검색할 수 있습니다.
관리자 비밀번호를 먼저 열면 검색된 데이터를 수정 화면으로 불러오거나 삭제할 수 있습니다.
삭제는 Supabase `delivery_records_delete` 정책이 적용되어 있어야 온라인 데이터에서도 삭제됩니다.

## 데이터 저장 구조

- 기본은 브라우저 로컬 저장소에 저장됩니다.
- GitHub Pages는 이 앱 화면을 배포하는 용도입니다.
- 온라인 데이터 보관은 Supabase가 담당합니다.
- 현재 앱에는 기본 Supabase 연결값이 포함되어 있어 사이트 접속 시 데이터를 자동으로 불러옵니다.
- Supabase에는 소모품/장비 데이터가 `delivery_records` 한 테이블에 저장되고, `asset_type` 컬럼으로 구분됩니다.
- 장비와 소모품의 관련 묶음은 `relation_group` 컬럼으로 저장됩니다.

## Supabase 연결

1. Supabase 프로젝트를 생성합니다.
2. SQL editor에서 `supabase-schema.sql` 내용을 실행합니다.
3. 앱에는 기본 Project URL과 Public Key가 들어가 있습니다.
4. `온라인 저장`을 누르면 현재 로컬 데이터가 `delivery_records` 테이블에 저장됩니다.
5. 다른 PC나 브라우저에서 사이트에 접속하면 데이터를 자동으로 불러오며, 필요하면 `온라인 불러오기`를 눌러 다시 가져옵니다.

현재 관리자 잠금은 앱 화면 안에서만 동작하는 간단한 비밀번호 방식입니다. 더 강한 보안이 필요하면 Supabase 로그인 기능을 붙이는 것이 좋습니다.

운영 단계에서는 회사 계정 단위 로그인을 붙이고, row level security 정책을 사용자 인증 기준으로 좁히는 것이 좋습니다.

## 대량 입력 컬럼

컬럼명은 아래와 같이 유지해 주세요.

- 분류
- 관리유형
- 납품일
- 병원명
- 담당자명
- 연결그룹
- 대표제품명
- 세부모델/규격
- 수량
- 단가
- LOT번호
- 유효기간만료일
- 넣은시점
- 품목코드
- 기타작성칸

대표제품명은 큰 묶음으로 관리하고, 모델 차이는 세부모델/규격 또는 기타작성칸에 적는 방식입니다.
`분류` 값은 `소모품` 또는 `장비`로 입력합니다.
`담당자명`은 나중에 누가 입력한 자료인지 확인하기 위한 값입니다.
`연결그룹`은 선택 입력이며, 같은 그룹명을 가진 장비/소모품은 분석 화면에서 연결 이력으로 함께 확인할 수 있습니다.

`delivery-expiry-manager` 폴더 전체를 GitHub 저장소에 올리면 됩니다.

- `styles.css`: 화면 디자인
- `app.js`: 입력, 분석, 내보내기, Supabase 동기화 기능
- `supabase-schema.sql`: 온라인 데이터 저장소 테이블 생성용 SQL
- `README.md`: 사용 설명

배포는 GitHub Pages로 하고, 데이터 보관은 Supabase로 하는 구조입니다.
